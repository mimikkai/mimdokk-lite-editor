# Add HTTP URL support for template loading in UI and API

**Date:** 2026-03-25 23:30
**Files:** server/routes/templates.ts, src/components/TemplateUploader.tsx
**Severity:** low

## Problem

No way to load a .docx template from an HTTP/HTTPS URL — only file upload was supported.
Users wanted to reference templates stored on remote storage (e.g. S3) directly by URL.

## Root Cause

The feature was missing from both ends:
- API: POST /api/templates only accepted `{ name, data }` (base64), not `{ name, url }`
- UI: TemplateUploader only had file input, no URL input

## Solution

**API** (`server/routes/templates.ts`): Accept `{ name, url }` in addition to `{ name, data }`.
When `url` is provided, the server fetches it with `fetch()`, converts to base64, stores in SQLite.
Validation: URL must start with "http", fetch errors return 502 with descriptive message.

**UI** (`src/components/TemplateUploader.tsx`): Added URL input field + Load button between
existing OR dividers. On submit, fetches URL client-side via `fetch()`, extracts filename
from the URL path, creates a synthetic `File` object, and calls the existing `onUpload` handler.
Supports Enter key to trigger load. Shows inline error if fetch fails.

## Prevention

- When adding new ingestion methods, always add both UI and API support simultaneously
- Server-side URL fetching is useful to avoid CORS issues on remote URLs (client-side fetch
  may fail for URLs that don't have permissive CORS headers — in that case the user can use
  the API to load the template server-side instead)
